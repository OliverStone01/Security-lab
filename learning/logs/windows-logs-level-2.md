# Windows logs - level 2: Core event categories

This file explains the main categories of Windows event logs and what type of activity they record. These categories help analysts decide where to look during an investigation.

-----

### Application

**What's recorded:**
- Events generated by an applications.
- Software crashes.
- Application errors.
- Application warnings.

**Common questions this type of log answers:**
- Did an application fail?
- Did software crash unexpectedly?
- Is a service not working correctly?

**Sources:**
- Browsers
- Databases
- Third-party tools

-----

### System

This log is used when investigating availability and stability

**What's recorded:**
- Operating system activity.
- Driver issues.
- Service start/stop events.
- Hardware related issues.

**Common questions this type of log answers:**
- Did the system boot correctly?
- Did a service fail to start?
- Did the system crash or restart?

-----

### Security

This log is the most important for SOC and security analysts.

**What's recorded:**
- Authentication events.
- Account changes.
- Privilage use.
- Security policy enforcement.

**Common questions this type of log answers:**
- Who logged in?
- Did a loging fail?
- Was admin access used?
- Were accounts created or modified?

-----

### Setup

This log helps explain why behavior changes after updates.

**What's recorded:**
- Operating system installation events.
- Updates and patches.
- Feature installations.

**Common questions this type of log answers:**
- Was an update applied?
- Did an installation fail?
- Was the system recently modified?

-----

### Forwarded events

This is mainly used in enterprise and SOC environments.

**What's recorded:**
- Events forwarded from other machines.
- Centralised logging environments.

**Common questions this type of log answers:**
- What is happening across multiple systems?
- Are multiple hosts affected by the same issue?
